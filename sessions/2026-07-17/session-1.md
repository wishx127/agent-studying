## 学习内容

### B.15 API 安全

**学生初始理解**:

- 能大致解释 XSS、CSRF、SQL 注入
- 能判断缺少认证凭证返回 401、已认证但无权限返回 403
- 认为 CSRF 可通过 Token 防御
- 初始认为前端输入校验、后端数据转义可以作为 SQL 注入主要修复方式

**讲解方式**:

- 区分 XSS 的浏览器脚本执行与 CSRF 的伪造状态变更请求
- 对比 Cookie 自动携带凭证与前端主动添加 Bearer Header 的 CSRF 风险
- 强调 CSRF 攻击的是请求意图，不是身份认证本身
- 区分输入格式校验、参数化查询、HTML 输出编码的职责
- 讲解网关层 + 认证服务 + 业务服务的分层限流，以及 IP/账号双维度防刷
- 介绍 QPS、平均延迟、平均并发之间的关系，并说明多实例需要共享限流状态

## 理解检验

- 能识别存储型 XSS，并提出输出转义方向；经追问理解应按输出上下文编码
- 能理解 Cookie 场景下 CSRF Token 的必要性，以及 Bearer Header 场景的风险重点转向 XSS
- 能识别字符串拼接 SQL 会改变 SQL 语义；经纠正后理解参数化查询是核心防线，Zod 不是替代方案
- 能提出网关按账号和 IP 限流，并理解账号锁定应短时、可恢复，避免锁定型 DoS
- 能正确理解 QPS = 请求数 / 秒数，以及平均并发数 ≈ QPS × 平均延迟
- 能判断 4 个实例各自本地限流 100 QPS 会放大到约 400 QPS，并理解共享 Redis 状态用于集群级聚合计数
