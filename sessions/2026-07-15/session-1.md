## 学习内容

### B.13 类型安全与运行时校验

**学生初始理解**:

- 类型检查可以防止数据类型错误
- Zod 主要负责格式校验
- 客户端传来错误 JSON 时，应该先由前端拦截

**讲解方式**:

- 区分 TypeScript 编译时类型检查与运行时数据校验
- 说明 `as User` 是类型断言，不会验证真实请求数据
- 强调前端校验用于用户体验，后端必须把所有外部输入视为不可信
- 区分 JSON 语法解析、Schema 输入契约校验和业务规则校验
- 介绍用 `z.infer<typeof Schema>` 从运行时 Schema 推导 TypeScript 类型，避免重复维护

**理解检验**:

- 能正确判断 `safeParse` 才会真正验证运行时数据，`as User` 只影响编译期
- 在订单场景中，能识别负数属于输入约束、库存不足属于业务服务校验
- 起初认为字符串数量只需要前端校验，经过追问后纠正为后端也必须校验
- 能理解双重校验和 Schema 作为类型单一来源的价值

**达成掌握程度**: L1
